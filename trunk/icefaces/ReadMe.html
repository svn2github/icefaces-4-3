Contents

Both Binary and Source Bundles are available for download. The contents of each bundle type vary, as described below.


Binary Bundle


The binary bundle comes with the ICEfaces libraries pre-compiled and the sample applications pre-built to run on "Servlet-profile" platforms, such as Tomcat. The binary bundle also includes all the source files for the sample applications, but not the ICEfaces libraries themselves. 

The icefaces directory of the binary bundle contains:
  • docs
    • The ICEfaces API and TLD documentation.
      • ace - JavaDocs and TLD docs for the ACE Components (api and tld).
      • compat - JavaDocs and TLD docs for the ICEfaces 1.x compatibility APIs (compat/api), and the ICE Components (compat/comps/api, compat/comps/tld).
      • core - JavaDocs and TLD docs for the core framework APIs and icecore tags.
  • lib
    • The ICEfaces libraries along with any possibly required 3rd-party run-time libraries.
      • icefaces.jar - The core ICEfaces framework.
      • icepush.jar - The ICEpush library, provides push functionality to the ICEfaces framework (optional).
      • icefaces-ace.jar - The ACE Components (ICEfaces Advanced Components) library (optional).
      • icefaces-compat.jar - ICEfaces compatibility library that includes the ICE Components and a set of ICEfaces 1.x core APIs to ease migration of existing ICEfaces 1.x applications to ICEfaces 3. (optional).
      • version-licenses.html - Version and license information for all included libraries.
  • licenses
    • Summary of legal licenses used by ICEfaces and 3rd party libraries.
  • samples
    • ace/themes
      • A set of jQuery ThemeRoller themes that have been converted to work with the ACE Components.
      • See the ACE ThemeRoller Support Wiki topic for details on how to configure and use these in your application.
    • compat
      • ICEfaces 1.x sample applications that have been ported to ICEfaces 3 using the ICE Components and the core compatibility APIs.
      • See the ICE Components and Compatibility topics in the ICEfaces Wiki for more information.
        • /auctionMonitor
    • core
      • ICEfaces core framework sample applications. Demonstrates using the ICEfaces framework with standard JSF h: components.
        • /auction
          • A simulated online auction application using only standard JSF components. Demonstrates core benefits of the ICEfaces framework, including the use of push to asynchronously update user's browsers when an auction bid occurs.
        • /portlets/chat
          • A simple ICEfaces chat portlet that demonstrates using ICEfaces and ICEpush in a portal environment.
    • dist
      • Contains all the sample app .wars prebuilt for "Servlet-profile" platforms, such as Tomcat.
    • showcase
      • ICEfaces Showcase sample application demonstrates key ICEfaces framework features and includes comprehensive demonstrations of both the ICE and ACE Components sets.
      • See the ICE Components and ACE Components topics in the ICEfaces Wiki for more information.
        • /showcase
        • /showcase-portlets

Source Bundle


The source bundle contains all the same libraries and applications as the binary bundle but also includes the complete source code for the ICEfaces libraries themselves, and additional required ICEfaces build-time libraries.  The source bundle does not contain any pre-built ICEfaces libraries or example .wars.

Use the "ant -p" command in the "icefaces" directory to see the available ant build targets.

The icefaces directory of the source bundle contains:
  • ace
    • Source files and resources for the ICEfaces Advanced Component Environment (ACE) and the ACE Components.
    • See the ACE Components topic of the ICEfaces Wiki for more information.
  • compat
    • Source files and resources for the ICE Components and the ICEfaces 1.x compatibility APIs.
    • See the Compatibility and ICE Components topics in the ICEfaces Wiki for more information.
  • core
    • Source files and resources for the ICEfaces core framework.
  • lib
    • The ICEfaces libraries are located here (after being built) along with any possibly required 3rd-party run-time and build-time libraries.
      • icefaces.jar - The core ICEfaces framework.
      • icepush.jar - The ICEpush library, provides push functionality to the ICEfaces framework (optional).
      • icefaces-ace.jar - The ACE Components library (optional).
      • icefaces-compat.jar - ICEfaces compatibility library that includes the ICE Components and a set of ICEfaces 1.x core APIs to ease migration of existing ICEfaces 1.x applications to ICEfaces 3. (optional).
      • version-licenses.html - Version and license information for all included libraries.
  • licenses
    • Summary of legal licenses used by ICEfaces and 3rd party libraries.
  • samples
    • ace/themes
      • A set of jQuery ThemeRoller themes that have been converted to work with the ACE Components.
      • See the ACE ThemeRoller Support Wiki topic for details on how to configure and use these in your application.
    • compat
      • ICEfaces 1.x sample applications that have been ported to ICEfaces 3 using the ICE Components and the core compatibility APIs.
      • See the ICE Components and Compatibility topics in the ICEfaces Wiki for more information.
        • /auctionMonitor
    • core
      • ICEfaces core framework sample applications. Demonstrates using the ICEfaces framework with standard JSF h: components.
        • /auction
          • A simulated online auction application using only standard JSF components. Demonstrates core benefits of the ICEfaces framework, including the use of push to asynchronously update user's browsers when an auction bid occurs.
        • /portlets/chat
          • A simple ICEfaces chat portlet that demonstrates using ICEfaces and ICEpush in a portal environment.
    • showcase
      • ICEfaces Showcase sample application demonstrates key ICEfaces framework features and includes comprehensive demonstrations of both the ACE and ICE Component sets.
      • See the ICE Components and ACE Components topics in the ICEfaces Wiki for more information.
        • /showcase
        • /showcase-portlets
